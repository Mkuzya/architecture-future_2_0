# Разделение системы на домены

## Домены системы

### 1. Домен клиники (Medical Domain)

**Описание:** Управление медицинскими данными, картами пациентов, историями болезни.

**Компоненты:**
- Legacy DWH (SQL Server) для медицинских данных
- Медицинские карты и истории болезни
- Результаты диагностических исследований

**Данные:**
- Медицинские карты
- Истории болезни
- Результаты исследований
- Данные по персоналу клиники
- Данные по инвентаризации

**Особенности:**
- Медицинские данные не включаются в витрину данных
- Требования к конфиденциальности и безопасности
- Соответствие требованиям регуляторов

### 2. Домен финтех (Fintech Domain)

**Описание:** Финансовые сервисы, банковские операции, кредиты, платежи.

**Компоненты:**
- Финтех-сервисы (Golang, Java)
- Финансовая история
- Счета
- Данные о кредитах
- Финансовая отчётность

**Данные:**
- Финансовая история клиентов
- Счета и платежи
- Кредитные данные
- Финансовая отчётность

**Особенности:**
- Банковская лицензия
- Требования PCI DSS
- Высокие требования к безопасности

### 3. Домен ИИ-сервисов (AI Domain)

**Описание:** Искусственный интеллект для анализа медицинских данных, диагностики.

**Компоненты:**
- ИИ-сервисы (Python)
- Модели машинного обучения
- Системы анализа данных

**Данные:**
- Агрегированные медицинские данные для анализа
- Результаты диагностики
- Модели и метрики

**Особенности:**
- Работа с агрегированными данными
- Высокие требования к производительности
- Необходимость масштабирования для обучения моделей

### 4. Домен аналитики (Analytics Domain)

**Описание:** Витрина данных, портал самообслуживания, BI-система.

**Компоненты:**
- Витрина данных (Data Mart)
- Портал самообслуживания
- BI система (Power BI)
- ETL сервисы

**Данные:**
- Агрегированные данные для аналитики
- Отчёты и дашборды
- Метаданные для создания отчётов

**Особенности:**
- Не содержит медицинских карт и историй болезни
- Быстрый доступ к данным
- Самообслуживание для бизнес-пользователей

### 5. Домен интеграции (Integration Domain)

**Описание:** Интеграция между доменами и внешними системами.

**Компоненты:**
- API Gateway
- Шина данных (Event Bus)
- Интеграционные сервисы

**Данные:**
- События и сообщения
- API запросы и ответы
- Метаданные интеграций

**Особенности:**
- Обеспечивает коммуникацию между доменами
- Изоляция доменов друг от друга
- Поддержка различных протоколов

### 6. Домен будущих партнёров (Partners Domain)

**Описание:** Интеграция с фармацевтическими компаниями и производителями электроники.

**Компоненты:**
- API для партнёров
- Интеграционные адаптеры
- Управление партнёрскими данными

**Данные:**
- Данные от партнёров
- Каталоги продукции
- Интеграционные метаданные

**Особенности:**
- Гибкая интеграция новых партнёров
- Изоляция от основных доменов
- Масштабируемость для новых бизнесов

## Обоснование разделения

### Преимущества для бизнеса

**1. Ускорение вывода продуктов**
- Каждый домен может развиваться независимо
- Изменения в одном домене не влияют на другие
- Параллельная разработка в разных командах

**2. Снижение зависимости от DWH**
- Бизнес-логика вынесена из DWH в домены
- DWH используется только для хранения данных
- Новые функции добавляются без изменения DWH

**3. Возможность отдельного развития направлений**
- Финтех-направление развивается независимо
- ИИ-сервисы могут масштабироваться отдельно
- Новые партнёры интегрируются без влияния на существующие системы

**4. Масштабирование для новых бизнесов**
- Новые домены добавляются без изменения существующих
- Изоляция данных и логики по доменам
- Гибкая архитектура для роста

**5. Улучшение производительности**
- Каждый домен оптимизируется независимо
- Распределение нагрузки между доменами
- Масштабирование отдельных доменов по потребности

### Технические преимущества

**1. Независимость развёртывания**
- Каждый домен развёртывается отдельно
- Откат изменений в одном домене не влияет на другие
- Непрерывная интеграция и доставка по доменам

**2. Изоляция данных**
- Данные каждого домена изолированы
- Снижение рисков утечек данных
- Упрощение управления доступом

**3. Масштабируемость**
- Каждый домен масштабируется независимо
- Оптимизация ресурсов под конкретные домены
- Гибкое управление нагрузкой

**4. Упрощение поддержки**
- Чёткие границы ответственности
- Упрощённая диагностика проблем
- Независимое тестирование доменов

## Связь с бизнес-сценариями

### Поддержка клиники
- Домен клиники обеспечивает работу медицинских сервисов
- Интеграция с ИИ-доменом для диагностики
- Изоляция медицинских данных от аналитики

### Поддержка финтех-сервисов
- Независимый домен финтех для банковских операций
- Интеграция с аналитикой для финансовой отчётности
- Изоляция финансовых данных

### Поддержка ИИ-сервисов
- Отдельный домен для ИИ-моделей
- Доступ к агрегированным данным
- Масштабирование для обучения моделей

### Интеграция новых партнёров
- Отдельный домен для партнёров
- Гибкая интеграция без изменения основных доменов
- Масштабирование для новых бизнесов
