# Технологический радар

## Текущие технологии

### Adopt (Используем)

**Microsoft SQL Server 2008**
- Применение: Legacy DWH для медицинских данных
- Бизнес-сценарии: Хранение медицинских карт и историй болезни
- Статус: Текущее использование, планируется миграция

**Power BI**
- Применение: BI система для визуализации и аналитики
- Бизнес-сценарии: Создание отчётов и дашбордов
- Статус: Продолжаем использовать, расширяем функциональность

**Python**
- Применение: ИИ-сервисы для анализа медицинских данных
- Бизнес-сценарии: Диагностика, анализ данных
- Статус: Продолжаем использовать, масштабируем

**Golang, Java**
- Применение: Финтех-сервисы
- Бизнес-сценарии: Банковские операции, кредиты, платежи
- Статус: Продолжаем использовать

**Apache Camel**
- Применение: Шина данных для интеграции
- Бизнес-сценарии: Коммуникация между сервисами
- Статус: Заменяем на современное решение

**PowerBuilder**
- Применение: Клиентский интерфейс для операторов
- Бизнес-сценарии: Работа с медицинскими данными
- Статус: Заменяем на веб-приложение

## Предлагаемые технологии

### Trial (Пробуем)

**Облачные платформы (AWS, Azure, Yandex Cloud)**
- Применение: Инфраструктура для всех доменов
- Бизнес-сценарии: Масштабирование для новых бизнесов, снижение затрат
- Обоснование: Гибкое масштабирование, снижение капитальных затрат

**Data Mesh архитектура**
- Применение: Организация данных по доменам
- Бизнес-сценарии: Независимое развитие направлений, интеграция новых бизнесов
- Обоснование: Поддержка доменной архитектуры, масштабируемость

**Kubernetes**
- Применение: Оркестрация контейнеров
- Бизнес-сценарии: Масштабирование сервисов, независимое развёртывание доменов
- Обоснование: Автоматическое масштабирование, управление ресурсами

**PostgreSQL / ClickHouse**
- Применение: Витрина данных, аналитические БД
- Бизнес-сценарии: Быстрая подготовка отчётности
- Обоснование: Высокая производительность для аналитики

### Assess (Оцениваем)

**Apache Kafka**
- Применение: Замена Apache Camel, event streaming
- Бизнес-сценарии: Асинхронная коммуникация между доменами
- Обоснование: Масштабируемость, надёжность, поддержка event-driven архитектуры

**Terraform**
- Применение: Infrastructure as Code
- Бизнес-сценарии: Автоматизация развёртывания инфраструктуры
- Обоснование: Воспроизводимость, версионирование инфраструктуры

**Apache Airflow**
- Применение: Оркестрация ETL процессов
- Бизнес-сценарии: Автоматизация загрузки данных в витрину
- Обоснование: Управление сложными ETL пайплайнами

**React / Vue.js**
- Применение: Портал самообслуживания
- Бизнес-сценарии: Создание отчётов бизнес-пользователями
- Обоснование: Современный UI, быстрая разработка

**FastAPI / Spring Boot**
- Применение: API Gateway, микросервисы
- Бизнес-сценарии: Единая точка входа для всех API
- Обоснование: Высокая производительность, простота разработки

### Hold (Откладываем)

**Microsoft SQL Server 2008**
- Применение: Legacy DWH
- Статус: Постепенная миграция, временно оставляем для медицинских данных
- Обоснование: Медицинские данные не используются в аналитике, можно мигрировать позже

**PowerBuilder**
- Применение: Клиентский интерфейс
- Статус: Заменяем на веб-приложение
- Обоснование: Устаревшая технология, сложность поддержки
